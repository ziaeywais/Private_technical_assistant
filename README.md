# Private Technical Assistant

Local RAG for your technical docs. Point it at a folder of PDFs, ask questions in plain English, get answers with source citations. Nothing goes to any API.

I built this because I got tired of uploading sensitive manuals to ChatGPT. Everything runs on your machine through Ollama.

---

## How it works

Search is hybrid, it runs both semantic search (Chroma) and BM25 keyword search in parallel, then merges the results. In practice this catches things that pure vector search misses, especially when you're searching for specific part numbers, error codes, that kind of thing.

There's an optional cross-encoder reranker you can toggle on if retrieval quality matters more to you than speed. For multi-turn conversations it rewrites follow-up questions before retrieval so "what about the second one?" actually finds the right chunk.

There's also a relevance threshold, if nothing in your docs is close enough to the question it just says so instead of making something up.

## What you need

- Python 3.10+
- [Ollama](https://ollama.com) installed and running
- The `nomic-embed-text` and `llama3.2` models (pulled via Ollama)

## Getting started

```bash
git clone git clone https://github.com/ziaeywais/Private_technical_assistant.git

cd Private-technical-assistant

python3 -m venv .venv
source .venv/bin/activate  # Windows: .venv\Scripts\activate

pip install -r requirements.txt
```

Pull the models if you haven't already:

```bash
ollama pull nomic-embed-text
ollama pull llama3.2
```

Drop your PDFs into a `data/` folder, then build the index:

```bash
mkdir data
python ingestion.py
```

Then start the app:

```bash
streamlit run app.py
```

Goes up at [http://localhost:8501](http://localhost:8501).

## Settings

Everything's adjustable from the sidebar at runtime:

- **Semantic search type** : `similarity` vs `mmr`. MMR tries to return more diverse chunks, useful if your docs are repetitive.
- **Semantic k / Keyword k** : how many chunks to pull from each search. Set keyword k to 0 to turn BM25 off entirely.
- **Final chunks to LLM** : the merged pool gets trimmed down to this before hitting the model.
- **Guardrail threshold** : relevance cutoff. Raise it if you're getting irrelevant answers, lower it if it's refusing too much.
- **Query rewriting** : recommended on if you're having real back-and-forth conversations.
- **Reranking** : better retrieval quality, ~1-2s slower per query. First time you enable it, downloads about 80 MB.
- **Source filter** : restrict search to a single PDF.
- **Debug mode** : shows the actual chunks that got passed to the model, useful when something's off.

---

## Adding or removing PDFs

Just edit the `data/` folder and re-run `python ingest.py`. It rebuilds everything from scratch, there's no incremental update.

---

## Things that might go wrong

**`Chroma DB not found`** , run `python ingest.py` before starting the app.

**Ollama connection refused** , Ollama needs to be running (`ollama serve` in another terminal).

**Reranker fails to load** , make sure `sentence-transformers` is installed: `pip install sentence-transformers`

**Runs out of memory during ingest** , lower the `BATCH_SIZE` constant at the top of `ingest.py`. It's 100 by default, try 25 or 50.
